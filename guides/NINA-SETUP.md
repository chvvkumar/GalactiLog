# N.I.N.A. Setup Guide

## Overview

GalactiLog reads data from three sources:

1. **FITS file headers** -- Metadata embedded in every captured frame by N.I.N.A. (always available)
2. **ImageMetaData.csv** -- Per-frame quality and equipment metrics generated by the [Session Metadata](https://github.com/tcpalmer/nina.plugin.sessionmetadata) plugin
3. **WeatherData.csv** -- Environmental data generated by the Session Metadata plugin (requires a connected weather source)

Only FITS files are required. The CSV files are optional but unlock more detailed per-frame metrics.

## Required: FITS Output Settings

N.I.N.A. writes most of these headers by default.

| FITS Header | Description | Used For |
|-------------|-------------|----------|
| `OBJECT` | Target name (e.g., "M31", "NGC 7000") | Target resolution via SIMBAD |
| `IMAGETYP` | Frame type: LIGHT, DARK, FLAT, BIAS | Frame classification, calibration filtering |
| `EXPTIME` | Exposure duration in seconds | Integration time calculations |
| `FILTER` | Optical filter name (e.g., "Ha", "OIII", "L") | Filter usage analytics, palette display |
| `INSTRUME` | Camera model | Equipment tracking |
| `TELESCOP` | Telescope/optics | Equipment tracking |
| `CCD-TEMP` | Sensor temperature in Celsius | Environmental monitoring |
| `GAIN` | Camera gain setting | Equipment metrics |
| `DATE-OBS` | Observation timestamp (ISO 8601) | Session grouping, timeline |
| `HFR` | Half-Flux Radius | Focus quality metric |
| `FWHM` / `MEANFWHM` | Full Width at Half Maximum | Star quality metric |
| `ECCENTRICITY` / `ELLIPTICITY` | Star elongation | Guiding/tracking quality |

### File Format

- Save as FITS format (`.fits`, `.fit`, or `.fts` extensions)
- GalactiLog reads the primary HDU (extension 0) headers
- All standard N.I.N.A. FITS output settings work without modification

### File Naming

GalactiLog can extract HFR from N.I.N.A.'s default filename pattern. If your filenames include the HFR value (e.g., `M31_Light_Ha_300s_1.56HFR_2025-01-15.fits`), it will be parsed as a fallback when the FITS header does not contain an HFR value.

## Optional: Session Metadata Plugin

The [Session Metadata](https://github.com/tcpalmer/nina.plugin.sessionmetadata) plugin generates `ImageMetaData.csv` and `WeatherData.csv` with per-frame metrics. These are not generated by N.I.N.A. itself.

### Installation

Install from the N.I.N.A. plugin manager: **Plugins**, search for "Session Metadata", and install.

### Configuration

Open **Plugins > Session Metadata** and configure:

| Setting | Default | Description |
|---------|---------|-------------|
| Plugin Enabled | Off | Must be enabled for CSV generation |
| CSV Enabled | On | Output CSV format (required for GalactiLog) |
| Non-Lights Enabled | Off | Enable to also log metadata for DARK, FLAT, BIAS frames |
| Weather Metrics Enabled | Off | Enable to generate WeatherData.csv (requires a connected weather source) |
| Metadata Output Directory | (empty) | Leave empty to write CSVs alongside your FITS files (recommended) |

File base names are customizable but GalactiLog expects the defaults (`ImageMetaData`, `WeatherData`). The plugin writes metadata on every image save event.

### ImageMetaData.csv

One row per captured frame with quality and equipment metrics:

| Metric | Description |
|--------|-------------|
| HFR | Half-Flux Radius |
| HFR Stdev | Standard deviation of HFR across detected stars |
| FWHM | Full Width at Half Maximum (requires Hocus Focus plugin) |
| Eccentricity | Star eccentricity (requires Hocus Focus plugin) |
| Detected Stars | Number of stars detected in the frame |
| Guiding RMS | Total guiding error (pixels and arcseconds) |
| Guiding RMS RA/Dec | Per-axis guiding error (pixels and arcseconds) |
| ADU Mean/Median/Stdev/Min/Max | Pixel value statistics |
| Camera Temp | Actual and target sensor temperature |
| Gain / Offset | Camera gain and offset |
| Focuser Position | Focuser step position |
| Focuser Temperature | Temperature at focuser |
| Rotator Position | Camera rotation angle |
| Pier Side | Mount pier side (East/West) |
| Airmass | Atmospheric airmass value |

### WeatherData.csv

One row per captured frame with environmental conditions (requires Weather Metrics Enabled and a connected weather source):

| Metric | Description |
|--------|-------------|
| Temperature | Ambient temperature |
| Humidity | Relative humidity percentage |
| Dew Point | Dew point temperature |
| Pressure | Atmospheric pressure |
| Wind Speed / Direction / Gust | Wind conditions |
| Cloud Cover | Cloud coverage percentage |
| Sky Quality | Sky quality meter reading (mag/arcsec^2) |
| Sky Brightness | Sky brightness |
| Sky Temperature | Sky temperature |

### Weather Sources

Any weather source compatible with N.I.N.A. will work:

- **OpenWeatherMap Plugin** -- Free API-based weather data (requires API key)
- **Pegasus Ultimate Powerbox** -- Integrated environmental sensors
- **AAG CloudWatcher** -- Cloud and rain detection
- **Davis Instruments** -- Weather station hardware
- **Other ASCOM weather drivers** -- Any ASCOM-compatible weather station

### Directory Placement

CSV files must be in the same directory as the FITS files they describe. Leave the output directory setting empty (the default) to achieve this.

```
/astro_incoming/
  M31/
    2025-01-15/
      ImageMetaData.csv          <-- Generated by Session Metadata plugin
      WeatherData.csv            <-- Generated if weather enabled
      M31_Light_Ha_300s_001.fits
      M31_Light_Ha_300s_002.fits
```

## Recommended N.I.N.A. Plugins

| Plugin / Component | Type | Metrics for GalactiLog |
|-------------------|------|----------------------|
| **N.I.N.A. Core** | Built-in | FITS headers (object, exposure, filter, camera, telescope, gain, sensor temp, date) |
| **N.I.N.A. HFR/Star Analysis** | Built-in | HFR, FWHM, eccentricity, detected stars (via FITS headers) |
| **[Session Metadata](https://github.com/tcpalmer/nina.plugin.sessionmetadata)** | Plugin | ImageMetaData.csv and WeatherData.csv -- per-frame quality metrics, equipment data, and environmental conditions |
| **PHD2 Guiding** | External | Guiding RMS total/RA/Dec (captured by Session Metadata plugin) |
| **N.I.N.A. Internal Guider** | Built-in | Guiding RMS (captured by Session Metadata plugin) |
| **Hocus Focus** | Plugin | FWHM and eccentricity values (captured by Session Metadata plugin) |
| **OpenWeatherMap** | Plugin | Weather data source for Session Metadata plugin |
| **Weather Station** (ASCOM) | Hardware | Weather data source for Session Metadata plugin |

## Directory Structure

GalactiLog scans recursively and works with any directory structure. A typical N.I.N.A. layout:

```
/astro_incoming/                        <-- GALACTILOG_FITS_HOST_PATH
  M31 - Andromeda Galaxy/
    2025-01-15/
      M31_Light_Ha_300s_001.fits
      M31_Light_Ha_300s_002.fits
      M31_Dark_300s_001.fits
      ImageMetaData.csv
    2025-01-20/
      M31_Light_OIII_300s_001.fits
      ImageMetaData.csv
  NGC 7000/
    2025-02-10/
      NGC7000_Light_SII_300s_001.fits
      ImageMetaData.csv
      WeatherData.csv
```

Scans for `.fits`, `.fit`, and `.fts` files (case-insensitive). Subdirectory depth and naming don't matter. Calibration frames are detected by the `IMAGETYP` header. CSV files must be alongside the FITS files they describe.
